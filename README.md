# Part 3 – Query Logs Using Kusto Query Language (KQL)

Failed login attempts were analyzed using KQL queries within Azure Log Analytics.

## KQL Query

```kusto
AppServiceConsoleLogs
| where ResultDescription contains "LOGIN_FAILED"
| summarize FailedAttempts = count() by bin(TimeGenerated, 5m)
| where FailedAttempts > 5
| order by TimeGenerated desc
```

## Query Explanation

The query performs the following actions:

1. Searches the AppServiceConsoleLogs table.
2. Filters entries containing the text LOGIN_FAILED.
3. Groups failed login attempts into five-minute intervals.
4. Counts the number of failures during each interval.
5. Identifies intervals containing more than five failed login attempts.
6. Sorts results from newest to oldest.

This query can be used to detect potential brute-force login attacks against the application.

# Part 4 – Create an Alert Rule

An Azure Monitor Alert Rule was created using the Log Analytics Workspace as the scope.

The alert was configured to identify excessive failed login attempts and notify administrators through an Action Group.

## Alert Configuration

| Setting                 | Value                   |
| ----------------------- | ----------------------- |
| Scope                   | Log Analytics Workspace |
| Detection Method        | Custom Log Search       |
| Evaluation Frequency    | 1 Minute                |
| Aggregation Granularity | 5 Minutes               |
| Severity                | 3                       |
| Action Group            | Email Notification      |

The alert is triggered whenever the system detects more than five failed login attempts within a five-minute period.
